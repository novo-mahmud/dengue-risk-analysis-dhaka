1. Introduction

This report presents an independent investigation into the risk factors driving Dengue fever transmission in Dhaka, Bangladesh. While public perception often links disease spread to housing quality (slums vs.developed areas), this study utilizes statistical inference and machine learning to validate the true drivers of infection.

Research Question: Does the built environment (Housing Type) determine Dengue risk, or is geographic location (Hotspots) the primary predictor?

2. Methodology

2.1 Tools & Dataset

  • Language: R (tidyverse, randomForest)
  • Dataset: dengue_dataset.csv (N = 1,000 patients)
  • Approach:
    1. Binary Logistic Regression (GLM) for Odds Ratios.
    2. Random Forest for Variable Importance validation.
    3. Administrative Aggregation (Thana Level) for policy insights.

2.2 Data Cleaning & Preprocessing

We begin by isolating the variables of interest and removing medical leakage (blood test results). This ensures our model predicts risk, not diagnosis.

# Load Libraries
if(!require("tidyverse")) install.packages("tidyverse")
if(!require("randomForest")) install.packages("randomForest")
library(tidyverse)
library(randomForest)

# Import Data
df <- read_csv("dengue_dataset.csv")

# Clean Data: Select relevant features & remove medical markers
df_clean <- df %>% 
  select(Age, Gender, Area, AreaType, HouseType, Outcome) %>%
  mutate(
    Gender = as.factor(Gender),
    AreaType = as.factor(AreaType),
    HouseType = as.factor(HouseType),
    Outcome = as.factor(Outcome)
  )

# Set Reference Level for Outcome (1 = Infected)
df_clean$Outcome <- relevel(df_clean$Outcome, ref = "1")

4.2 Model 2: Geographic Hotspots (The Pivot)

Since infrastructure was not significant, we pivot to analyzing Location (Area). We set the most populous area as the baseline for stability.

# Automate Baseline Selection
area_counts <- table(df_clean$Area)
new_baseline <- names(sort(area_counts, decreasing = TRUE))[1]
df_clean$Area <- as.factor(df_clean$Area)
df_clean$Area <- relevel(df_clean$Area, ref = new_baseline)

# Run Geographic Model
geo_model <- glm(Outcome ~ Area, data = df_clean, family = "binomial")

# Calculate Odds Ratios
geo_odds <- exp(coef(geo_model))
geo_ci <- exp(confint(geo_model))
geo_results <- cbind(Odds_Ratio = geo_odds, CI_Lower = geo_ci[,1], CI_Upper = geo_ci[,2])

# Display Top Results

6. Administrative Level Analysis (Thana)

Finally, we aggregate the granular Area data into official Thanas to identify regional administrative risks.

# Merge and Aggregate
df_merged <- left_join(df_clean, thana_lookup, by = "Area")
thanx_stats_sorted <- df_merged %>%
  group_by(Thana) %>%
  summarise(
    Total_Patients = n(),
    Total_Cases = sum(as.numeric(Outcome)),
    Infection_Rate = (sum(as.numeric(Outcome)) / n()) * 100
  ) %>%
  arrange(desc(Infection_Rate))

6.1 Thana Risk Heatmap

plot_data_final <- thanx_stats_sorted %>% arrange(desc(Infection_Rate))
plot_data_final$Thana <- factor(plot_data_final$Thana, levels = plot_data_final$Thana)

7. Conclusion

This study successfully identified that Geographic Location (Thana) is the statistically significant predictor of Dengue risk, superseding housing type. Specifically, Sutrapur and Ramna Thanas demonstrate the highest infection rates. These findings suggest that vector control measures (drainage, fumigation) should be prioritized based on geographic clustering rather than solely on housing development status.
